Le programme Easy CHM apporte un vrai gain de temps par sa possibilité de créer automatiquement une TOC (Table des contenus) sous forme d'arborescence en plusieurs couches (Multi-niveaux) dès que l'utilisateur a choisi le point de départ de sa ligne de conduite.Toute la structure des dossiers et des fichiers qui découlent de ce point de départ est reproduite en temps qu' items TOC d'un fichier CHM, vous pouvez éditer/déplacer/supprimer ces items TOC. Avec cette possibilité de gain de temps, vous n'êtes plus obligés de toujours ajouter manuellement et un par un les items TOC. 
Comment modifier les items 'icônes' dans une TOC (Table des contenus)? Choisir un item TOC, cliquer-droit dessus et sélectionner'Propriété TOC', ici vous avez la possibilité de sélectionner une icône parmi 42 icônes différentes pour cet item TOC.
Comment modifier en lots les items 'icônes' dans une TOC? Sélectionner un item TOC, cliquer-droit dessus et sélectionner'Changer les icônes en lot...', faites les choix et pressez le bouton 'OK'
Souhaitez-vous posséder la version la plus récente? Cliquer simplement sur 'Mise à jour automatique...' dans le menu principal 'd'Aide', et le programme contrôlera directement dans le site web et s'il y a lieu chargera automatiquement la nouvelle version.
Par défaut Easy CHM crée automatiquement un fichier d'aide en html sensible au contexte, c'est à dire que tous les 'Id's-Contexte'sont automatiquement engendrés. Si vous êtes un développeur de software, s'il vous plait reprenez toutes les définitions des 'Id's-Contexte'à partir de ces deux fichiers : 'ecAlias.h' et 'ecMap.h';(Ces deux fichiers sont engendrés par Easy CHM, vous pourrez les trouver dans le dossier racine du projet. Ils ne doivent nullement être présentés en public).
Afin de réaliser un fichier CHM sans erreur de navigation, s'il vous plaît servez-vous de nom de fichiers et de dossiers exempt de certains caractères spéciaux tels que '&' or '#';  par exemple, un nom de fichier tel que '123#ebook.html' et '123&456.html' sera la cause d'erreur de navigation dans un CHM compilé.
En vue de réaliser un fichier CHM exempt d'erreur de navigation dans l'Index/TOC, s'il vous plaît utilisez également un nom de fichier ou de dossier exempt d'espace blanc en tête de dénomination, par exemple, un nom de fichier comme ' 123ebook.html'sera la cause d'une erreur de navigation dans dans le fichier CHM compilé. Si vous recontrez une erreur de navigation avec l'Index/TOC du CHM, s'il vous plaît pensez à contrôler le nom du fichier. 
Nécessité de faire apparaître les boutons 'Précédent' et 'Suivant'dans un CHM complilé? Cocher 'Créer un TOC binaire' dans l'étiquette 'Général' des 'Réglages CHM', ensuite cocher 'Suivant' et 'Précédent' dans l'étiquette 'Boutons' - Faire apparaître ces deux boutons requiert l'existance d'un cochage en 'Créer le TOC binaire'. 
S'il vous plaît, ne jamais utiliser un chemin local absolu dans vos fichiers '.html'. Un chemin local absolu tel que 'c:\mybook\myfile.htm' ou 'fichier://c:\mybook\myfile.htm' causera une erreur de navigation dans le CHM compilé, spécialement lorsque vous envoyez le CHM compilé vers d'autres utilisateurs ou vers des utilisateurs sur l'Internet qui n'ont pas un accès direct à votre PC -Ils ne peuvent voir correctement une image dans un CHM, si cette image est liée à un fichier dans votre propre disque dur.
Ne souhaitez pas indexer chaque fois manuellement les 'Paramètres CHM ' ! Il existe dans 'Paramètres du CHM' un 'Choix rapide d'options'  qui peut vous faciliter la fixation de tous vos préréglages. Après avoir imposé tous les réglages, vous pouvez créer un nouveau schéma, imposer un nom à ce schéma, et ensuite le sauvegarder. La fois suivante vous pouvez choisir les schémas nécessaires dans cette liste de schémas, et le programme chargera automatiquement tous les choix prédéfinis. Bien sûr, vous pouvez renommer ou supprimer le schéma si vous le désirez. 
Il est utile lorsque vous le souhaitez de prévoir des fichiers CHM en différents styles, par exemple, avec ou sans texte intégral de recherche, avec ou sans bouton 'Type de caractère', etc.
N'établissez pas de lien avec un fichier portant un nom tel que '3_4.ht1.jpg' ou '3_4.htm.jpg' dans votre fichier HTML, ces noms de fichiers risquent parfois de faire crasher le compilateur du CHM, et les images ne pourront pas apparaitre correctement. Il vaut mieux modifier '3_4.ht1.jpg' ou '3_4.htm.jpg' en  '3_4_htm.jpg'.   S'il vous plait, ne composez pas vos fichiers HTML en utilisant Ms WORD (Vous pouvez à la place travailler avec Ms FRONTPAGE pour céer vos fichiers HTML), il existe différents codes de LIAISON en fichiers HTML qui sont maintenus en Ms WORD, et certains codes Image  ne sont pas complètement admis par le Ms CHM SYSTEM, c'est à dire que parfois vous verrez apparaître un 'X' rouge en lieu et place d'une image dans le fichier CHM compilé. 
Si vous avez plusieurs fichiers .DOC et vous souhaitez les introduire dans le système fichier HTML en vue de créer un fichier CHM unique, s'il vous plait chargez à partir du site officiel de MS un ajoût pour Ms WORD dénommé 'Office2000 HTML Filter', et avec l'aide de cet ajoût vous pourrez sauvegarder des fichiers .DOC en fichiers Html compactes et propres... (Sauvegarde de toutes les balises requises en Html)
Les fichiers MS CHM ne sont acessibles par rien d'autres qu'un pilote local, s'il vous plait assurez-vous  que le fichier CHM est bien copié ou chargé sur votre PC local avant d'essayer d'ouvrir un fichier CHM pour éviter cet avis d'erreur 'Page impossible à présenter'.
Il existe trois manières de renommer les items de TOC ou d'INDEX , sélectionner un item et double-cliquer dessus, sélectionner un item et Presser F2, ou encore sélectionner un item et cliquer sur 'Editer' dans le menu déroulant.
