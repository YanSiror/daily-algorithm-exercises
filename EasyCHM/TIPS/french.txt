Le programme Easy CHM, apporte un vrai gain de temps par sa possibilité de créer automatiquement une TOC (Table des matières) sous forme d'arborescence en plusieurs couches (Multi-niveaux) dès que l'utilisateur a choisi le répertoire de départ. Toute la structure des répertoires et des fichiers qui découlent de ce point de départ est reproduite en temps qu' éléments de TOC dans le fichier CHM, vous pouvez bien entendu éditer/déplacer/supprimer ces éléments. Avec cette caractéristique, vous n'êtes plus obligés de toujours ajouter manuellement et un par un les éléments de la TOC. 
Comment modifier les icônes des éléments dans une TOC (Table des matières) ? Choisir un élément, cliquer-droit dessus et sélectionner'Propriétés de la TOC', vous avez à ce moment la possibilité de sélectionner une icône parmi 42 icônes différentes pour cet élément.
Comment modifier par lots les icônes des éléments dans une TOC ? Sélectionner un élément, cliquer-droit dessus et sélectionner 'Changer les icônes par lots...', faites votre choix et cliquez le bouton 'OK'
Vous souhaitez posséder la version la plus récente ? Cliquer simplement sur 'Mise à jour automatique...' dans le menu principal "Aide", et le programme contrôlera directement sur le site web de l'auteur et s'il y a lieu chargera automatiquement la nouvelle version.
Par défaut Easy CHM crée automatiquement un fichier d'aide contextuelle en html, c'est à dire que tous les "IDs d'aide contextuelle" sont automatiquement générés. Si vous êtes développeur de programme, utilisez toutes les définitions des "IDs d'aide contextuelle" à partir dans ces deux fichiers : 'ecAlias.h' et 'ecMap.h' ; (Ces deux fichiers sont générés par Easy CHM, vous pourrez les trouver dans le répertoire racine du projet. Il n'est pas nécessaire de distribuer ces deux fichiers au destinataire du fichier CHM).
Afin de réaliser un fichier CHM sans erreur de navigation, servez-vous de noms de fichiers et de répertoires exempts de certains caractères spéciaux tels que '&' ou '#' ; par exemple, un nom de fichier tel que '123#ebook.html' et '123&456.html' occasionnera des erreurs de navigation dans un CHM compilé.
En vue de réaliser un fichier CHM exempt d'erreurs de navigation dans l'Index/TOC, utilisez également un nom de fichier ou de réperoire exempt d'espace en tête de dénomination, par exemple un nom de fichier comme '123ebook.html' sera la cause d'une erreur de navigation dans dans le fichier CHM compilé. Si vous recontrez une erreur de navigation avec l'Index/TOC du CHM, pensez à contrôler le nom du fichier. 
Nécessité de faire apparaître les boutons 'Précédent' et 'Suivant' dans un CHM complilé ? Cocher 'Créer une TOC binaire' dans l'onglet 'Général' des 'Réglages CHM', ensuite cocher 'Suivant' et 'Précédent' dans l'onglet 'Boutons' - Pour faire apparaître ces deux boutons, il faut que 'Créer une TOC binaire' soit coché. 
Ne jamais utiliser un chemin local absolu dans vos fichiers '.html'. Un chemin local absolu tel que  'c:\monlivre\monfichier.htm' ou 'fichier://c:\monlivre\monfichier.htm' causera une erreur de navigation dans le CHM compilé, spécialement lorsque vous distribuez le CHM compilé vers d'autres utilisateurs ou vers des utilisateurs sur Internet qui n'ont pas un accès direct à votre PC - Ils ne peuvent en effet voir correctement une image dans un CHM, si cette image est liée à un fichier sur votre propre disque dur.
Vous ne souhaitez pas indexer chaque fois manuellement les 'Paramètres du CHM' ! Il existe dans 'Paramètres du CHM' une possibilité de choix rapide d'options qui vous facilitera les préréglages. Après avoir défini tous vos réglages, vous pouvez créer un nouveau schéma, donner un nom à ce schéma, et ensuite le sauvegarder. La prochaine fois, vous pourrez choisir le schéma qui convient dans cette liste, et le programme chargera automatiquement tous les réglages prédéfinis. Bien sûr, vous pouvez renommer ou supprimer un schéma si vous le désirez. 
Il est utile lorsque vous le souhaitez de prévoir des fichiers CHM en différents styles, par exemple, avec ou sans recherche de texte, avec ou sans bouton "Police", etc.
N'établissez pas de lien avec un fichier portant un nom tel que '3_4.ht1.jpg' ou '3_4.htm.jpg' dans votre fichier HTML, ces noms de fichiers risquent parfois de faire planter le compilateur CHM, et les images ne s'afficheront pas correctement. Il vaut mieux modifier '3_4.ht1.jpg' ou '3_4.htm.jpg' en '3_4_htm.jpg'. Ne créez pas vos fichiers HTML en utilisant MS Word (vous pouvez utiliser MS Frontpage pour les créer), il existe différents codes superflus dans les fichiers HTML qui sont générés sous Ms Word, et certains codes Image ne sont pas complètement supportés par le système Microsoft CHM, c'est à dire que parfois vous verrez apparaître un 'X' rouge en lieu et place d'une image dans le fichier CHM compilé. 
Si vous avez plusieurs fichiers .DOC et voulez les enregistrer comme fichiers HTML en vue de créer un fichier CHM, chargez à partir du site officiel de Microsoft un modificatif pour Ms Word nommé 'HTML Filter pour Office 2000', et grâce à cette modification, vous pourrez enregistrer des fichiers .DOC en fichiers Html compacts et propres... (sauvegarde de toutes les balises requises en Html)
Les fichiers MS CHM ne sont accessibles que par un pilote local, assurez-vous  que le fichier CHM est bien copié ou téléchargé sur votre PC avant d'essayer de l'ouvrir pour éviter cet avertissement 'La page ne peut être affichée'.
Il existe trois manières de renommer les éléments de TOC ou d'Index : sélectionner un élément et double-cliquer dessus, sélectionner un élément et presser F2 ou encore sélectionner un élément et cliquer sur 'Editer' dans le menu déroulant qui apparaît.
Petit glossaire : La TOC (ou Table de Contenus = Table des matières) est contenue dans un fichier qui définit les différents chapitres du fichier d'aide. Le fichier Index contient les mots-clés qui permettent de faire une recherche dans le fichier. Un fichier MAP contient les directives permettant de mettre en correspondance une partie d'application avec la rubrique correspondante du fichier d'aide. Un fichier Alias contient des raccourcis pour le nom d'une page du fichier d'aide (par exemple pour l'alias accueil : Accueil=html\intro.html). Un schéma est un fichier qui contient une configuration particuière de EasyCHM, ce qui permet de le configurer très rapidement en chargeant le fichier voulu .
