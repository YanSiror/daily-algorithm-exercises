Ein sehr zeitsparende Feature von Easy CHM ist, dass das Programm automatisch eine Mehr-Ebenenbaum für das Inhaltsverzeichnis aufbauen kann, wenn der Anwender ein Startverzecihnis gewählt hat. Alle Verzeichnisstrukturen und Dateien, die sich unter dem Startverzeichnis befinden, werden in das Inhaltsverzeichnis (TOC=Table of Contents) des CHM übernommen. Sie können diese Einträge editieren, verschieben oder löschen. Mit diesem zeitsparenden Feature müssen Sie nie einen TOC-Eintrag manuell einfügen.
Wie können Sie das Icon des Inhaltsverzeichnisses (TOC=Table of Contents) verändern? Wählen Sie ein TOC Eintrag aus, klicken Sie rechts and wählen Sie 'TOC Eigenschaften', um dann das Icon für diesen TOC Eintrag aus einer Liste von 42 verschiedenen Icons auszuwählen.  
Wie können Sie die Icons der TOC Einträge im Batch ändern? Wählen Sie ein TOC Eintrag aus, klicken Sie rechts und wählen Sie 'Wechsele Icons in Batch...' aus, stellen Sie die anderen Einstellungen ein und klicken Sie auf den 'Ok'-Button. 
Möchten Sie die neueste Version laden? Klicken Sie auf 'Automatisches Update...' im Hauptmenü unter 'Hilfe' und das Programm prüft automatisch auf der Webseite, ob eine neue Version vorhanden ist und führt einen Download der neuesten Version durch.
Als Standard legt Easy CHM autoamtisch eine Kontextsensitive HTML-Hilfe-Datei an. Dazu ist zu sagen, das alle CONTEXT-ID's automatisch erzeugt werden. Wenn Sie ein Softwareentwickler sind, dann verwenden Sie alle CONTEXT-IDs' Definitionen dieser zwei Dateien: 'ecAlias.h' und 'ecMap.h' (Diese zwei Dateien werden von Easy CHM erzeugt und Sie finden diese Dateien im Projektwurzelverzeichnis. (Sie brauchen diese zwei Dateien Ihren Anwender nicht zu veröffentlichen.)
Um eine CHM Datei ohne Navigationsfehler zu erstellen, verwenden Sie Datei- und Verzeichnisnamenohne die Sonderzeichen wie '&' oder '#', so erzeugen z.B. Dateinamen wie '123#ebook.html' und '123&456.html' einen Navigtaionsfehler in der komplierten CHM Datei.
Um eine CHM Datei ohne TOC-/Index-Navigationsfehler zu erzeugen, verwenden Sie bitte keine Datei- oder Verzeichnisnamen mit einem Leerzeichen am Anfang. So erzeugt z.B. eine Dateiname wie' 123ebook.html' einen Navigationsfeler in der kompilierten CHM Datei. 
Wenn Sie einen Navigationsfehler im TOC oder im Index der CHM Datei erhalten, dann prüfen Sie bitte den Dateinamen. 
Möchten Sie die zwei Buttons 'Previous' und 'Next' in der komplierten CDM Datei verwenden? Prüfen Sie die 'CHM Einstellungen', das 'Anlegen einer binären TOC' im Abschnitt 'Allgemein' und 'Next' und 'Previous' im Abschnitt 'Buttons' - Um diese zwei Buttons zu zeigen, muss 'Anlegen einer binären TOC' ausgewählt sein.
Bitte verwenden Sie nie absolute Verzeichnisse in Ihren .html-Dateien. Absolute Verzeichnisnamen wie 'c:\mybook\myfile.htm' oder 'file://c:\mybook\myfile.htm' erzeugen einen Navigationsfehler in der komplierten CHM Datei, insbesondere wenn Sie die kompliertee CHM zu einem anderen Anwender senden oder zu einem anderen Internet-User, der nicht direkt auf Ihren PC zugreifen kann - Dieser Anwender kann kein Bild in der CHM Daeien sehen.
Möchten Sie die 'CHM Einstellungen' immer manuel setzen? Es ist ein Schnellauswahl in den 'CHM Einstellungen'vorhanden, die Ihnen helfen kann, alle Ihre bevorzugten EInstellungen zu verwenden. Nachdem Sie alle Einstellungen gesetzt haben, können Sie ein neues Schema erzeugen, einen Schemanamen eingeben und dann abspeichern. Bei nächsten Mal können Sie das notwenidgen Schema auswählen aus einer Liste auswählen und das Programm liest alle vordefinieren EInstellungen automatisch. Natürlich können Sie das Schema umbennen oder löschen. Dies ist sehr hilfreich, wenn Sie CHM-Dateien mit verschiedenen Aussehen erzeugen möchten, z.B. mit oder ohne eine Volltextsuche, mit oder ohne Font-Button usw.
Erzeugen Sie in IHren HTML Dateinen keinen Verweis zu einer Datei wie '3_4.ht1.jpg' oder '3_4.htm.jpg', denn diese Dateinamen werden den CHM Compiler unter Umständen abbrechen lassen und die Bilder werden dann nicht angeziegt. Das beste ist es, den Namen '3_4.ht1.jpg' oder '3_4.htm.jpg' in '3_4_htm.jpg' zu ändern. 
Bitte erzeigen Sie keine HTML Dateien mit MS WORD (Sie können aber MS FRONTPAGE verwenden, um HTML Dateien zu erzeugen). In HTML Dateien von MS WORD sind viele unnötige Zeichen und einige Zeicenkodes werden nicht komplett vom CHM SYSTEM. Manchmal finden Sie ein rotes 'X' anstatt eines Bildes in der kompílierten CHM Datei..
Wenn Sie viele .DOC Dateien haben und diese als HTML Dateinen speichern möchten, um eine CHM Datei zu erzeugen, dann laden Sie bitte das Addin fürr MS WORD, das als 'Office2000 HTML Filter' bezeichnet wird, von der offiziellen MS Webseite. Mit diesem Addin können Sie .DOC Dateien kompakt und "sauber" als HTML Dateien speichern.
Auf MS CHM Dateien kann nicht zugriffen werden, ausser von einem loklame Laufwerk. Bitte vergewissern Sie sich, dass die CHM Datei kopiert oder auf Ihren lokalen PC geladen sit, bevor Sie versuchen eine CHM Datei zu öffnen, sonst erscheint der 'Page cannot be displayed' Fehler.
